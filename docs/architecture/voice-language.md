# R2D2 Voice Language

R2D2 communicates via characteristic beep sequences. This document describes
the vocabulary, the sound generation pipeline, and how the LLM integrates
with the voice system.

## Overview

```
LLM structured output
  { "speak": true, "intent": "task_complete", "emotion": "happy", "intensity": "high" }
          |
          v
  intent_mapper.py          <- maps intent/emotion/intensity to phrase list
          |
          v
  ["phrase_task_done", "phrase_happy"]
          |
          v
  voice_node.py             <- ROS2 node, subscribes to /r2d2/voice_intent
          |
          v
  aplay phrase_task_done.wav -> aplay phrase_happy.wav
          |
          v
  Speaker
```

## Sound hierarchy

### Phonemes (14 atomic sounds)

Generated by `scripts/synth.py` via FM synthesis.

| Phoneme           | Type    | fc (Hz) | fm (Hz) | Description              |
|-------------------|---------|---------|---------|---------------------------|
| chirp_hi_short    | chirp   | 1600    | —       | Short high beep           |
| chirp_lo_short    | chirp   | 680     | —       | Short low beep            |
| chirp_hi_long     | chirp   | 1400    | 18      | Long high beep, slight FM |
| blip_neutral      | chirp   | 920     | —       | Neutral mid blip          |
| alert_beep        | chirp   | 1950    | —       | Sharp warning beep        |
| sweep_up_fast     | sweep   | 520→1750| —       | Rising intonation         |
| sweep_up_slow     | sweep   | 420→1450| —       | Slow rise                 |
| sweep_down_fast   | sweep   | 1750→520| —       | Falling intonation        |
| sweep_down_slow   | sweep   | 1450→400| —       | Slow fall                 |
| sweep_up_rich     | sweep   | 550→1600| 12      | Rise with FM texture      |
| wobble_happy      | wobble  | 900     | 8       | Slow happy warble         |
| wobble_curious    | wobble  | 800→1100| 10      | Rising curious wobble     |
| wobble_sad        | wobble  | 720→480 | 5       | Slow descending warble    |
| trill_excited     | trill   | 1200    | 48      | Fast excited trill        |
| trill_alarm       | trill   | 1500    | 60      | Urgent alarm trill        |
| buzz_negative     | buzz    | 340     | 75      | Low buzzy negative        |
| hum_processing    | ambient | 740     | 11      | Processing hum            |

|-------------------------|--------------------------------------------------------------------|

| phrase_happy            | wobble_happy → trill_excited → chirp_hi_short                      |
| phrase_sad              | wobble_sad → sweep_down_slow                                       |
| phrase_excited          | trill_excited → sweep_up_fast → chirp_hi_short → trill_excited     |

## LLM integration

Include the following block in R2D2's main system prompt:

```
When R2D2 should react vocally, include a "voice" key in your JSON output:

{
  "voice": {
    "speak": true,
    "intent": "<intent>",
    "emotion": "<emotion>",
    "intensity": "<low|medium|high>"
  }
}

intent options:
  affirmative, negative, question, greeting, goodbye,
  task_complete, task_failed, thinking, warning, comment

emotion options:
  happy, sad, excited, scared, confused, curious, angry, neutral

intensity options: low, medium, high

Set speak=false if no vocal reaction is needed.
Use intent="comment" + emotion for pure emotional reactions.
Only output values from the lists above.
```

### Example LLM outputs

```json
// R2D2 just avoided an obstacle
{"voice": {"speak": true, "intent": "task_complete", "emotion": "excited", "intensity": "high"}}

// Stair detected (SRF02 triggered)
{"voice": {"speak": true, "intent": "warning", "emotion": "scared", "intensity": "high"}}

// User greeted R2D2
{"voice": {"speak": true, "intent": "greeting", "emotion": "happy", "intensity": "medium"}}

// No reaction needed
{"voice": {"speak": false}}
```

## ROS2 topics

| Topic                  | Type              | Direction  | Description                    |
|------------------------|-------------------|------------|--------------------------------|
| /r2d2/voice_intent     | std_msgs/String   | subscribe  | LLM voice output JSON          |
| /r2d2/voice_playing    | std_msgs/String   | publish    | Playback status                |

## Setup

Generate the `.wav` library on the Pi (one-time, after any synth.py change):

```bash
cd ~/r2d2_ws/src/r2d2_audio
pip install numpy
python3 scripts/synth.py
```

The `sounds/` directory is git-ignored (binary files). Add it to `.gitignore`:

```
src/r2d2_audio/sounds/
```

Build and run:

```bash
cd ~/r2d2_ws && colcon build --packages-select r2d2_audio
source install/setup.bash
ros2 run r2d2_audio voice_node
```

Manual test:

```bash
ros2 topic pub --once /r2d2/voice_intent std_msgs/String \
  '{"data": "{\"speak\": true, \"intent\": \"greeting\", \"emotion\": \"happy\", \"intensity\": \"high\"}" }'
```
